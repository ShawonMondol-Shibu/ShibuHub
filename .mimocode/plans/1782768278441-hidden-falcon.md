# ShibuHub Production-Grade Implementation Plan

> Transform localStorage prototype → full-stack e-commerce app with real auth, database, payments, and CRUD dashboard. Portfolio-ready for job applications.

## Target Stack
- **Auth**: BetterAuth (better-auth.com)
- **ORM**: Drizzle ORM
- **Database**: NeonDB (serverless PostgreSQL)
- **Payments**: Stripe
- **Deployment**: Vercel

## Phase 3: Database Schema (Drizzle + NeonDB) (~3h)

### Create:
- `.env.example` — DATABASE_URL, BETTER_AUTH_SECRET, STRIPE keys, NEXT_PUBLIC_APP_URL
- `drizzle.config.ts` — PostgreSQL config pointing to `db/schema.ts`
- `db/index.ts` — NeonDB connection with Drizzle
- `db/schema.ts` — Full schema (see below)
- `db/seed.ts` — Fetch from fakestoreapi.com, insert into NeonDB

### Schema tables:
| Table | Key columns |
|-------|-------------|
| `users` | id (uuid), name, email, emailVerified, image, phone, address, role (admin/manager/staff) |
| `categories` | id, name, slug, description, imageUrl |
| `products` | id, name, slug, description, price, compareAtPrice, sku, stock, images (jsonb), categoryId (FK), brand, model, isActive |
| `orders` | id, userId (FK), status (pending/processing/shipped/delivered/cancelled), total, shippingAddress (jsonb), stripeSessionId, stripePaymentIntentId |
| `order_items` | id, orderId (FK cascade), productId (FK), quantity, price, name, image |
| `reviews` | id, userId (FK), productId (FK), rating, comment |
| `favourites` | id, userId (FK), productId (FK) |
| `contact_messages` | id, name, email, phone, message, isRead |

### Add scripts to package.json:
```json
"db:generate": "drizzle-kit generate",
"db:push": "drizzle-kit push",
"db:seed": "tsx db/seed.ts",
"db:studio": "drizzle-kit studio"
```
